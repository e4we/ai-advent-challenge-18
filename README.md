# MCP Reminder Server

MCP-сервер напоминаний на Go. Позволяет Claude Desktop и Claude Code создавать, отслеживать и удалять напоминания — как одноразовые (через N минут), так и периодические (по cron-расписанию).

## Описание

Сервер реализует [Model Context Protocol](https://modelcontextprotocol.io) и предоставляет Claude четыре инструмента для работы с напоминаниями. Напоминания хранятся в SQLite-базе, фоновый планировщик автоматически их обрабатывает каждые 30 секунд.

При срабатывании напоминания сервер отправляет push-уведомление подключённым клиентам через стандартный механизм MCP-нотификаций (`notifications/resources/list_changed`). CLI-агент реагирует на уведомление без ввода пользователя — сам сообщает о сработавшем напоминании.

## Архитектурная схема

```
CLI-агент / Claude Desktop / Claude Code
        ↕  JSON-RPC (stdin/stdout)          ↑ notifications/resources/list_changed
┌──────────────────────────────────┐        │
│          MCP-сервер              │────────┘
│   (StdioTransport, главная       │
│    горутина, ждёт запросы)       │
│                                  │
│  Tools (handlers):               │
│   create_reminder                │
│   list_reminders    ─────────────┼──→ Storage (SQLite)
│   get_summary                    │         ↑
│   delete_reminder                │         │
└──────────┬───────────────────────┘         │
           │ OnFired коллбэк                  │
           │ AddResource("reminder://fired/…") │
           │                                  │
┌──────────▼──────────────┐                  │
│  Scheduler (горутина)   │──────────────────┘
│  тикер каждые 30 сек    │
│  checkAndFire()         │
│  reschedule()           │
└─────────────────────────┘
```

## Сборка

```bash
cd mcp-reminder
go build -o mcp-reminder .
```

На Windows:

```bash
go build -o mcp-reminder.exe .
```

## Подключение к Claude Desktop

Добавь в конфигурационный файл Claude Desktop:

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

```json
{
  "mcpServers": {
    "reminder": {
      "command": "/полный/путь/к/mcp-reminder"
    }
  }
}
```

На Windows:

```json
{
  "mcpServers": {
    "reminder": {
      "command": "C:\\путь\\к\\mcp-reminder.exe"
    }
  }
}
```

После изменения конфига перезапусти Claude Desktop.

## Подключение к Claude Code

Добавь в `.mcp.json` в корне проекта или в глобальный конфиг:

```json
{
  "mcpServers": {
    "reminder": {
      "command": "/полный/путь/к/mcp-reminder",
      "type": "stdio"
    }
  }
}
```

## Примеры использования

Фразы, которые можно говорить Claude после подключения:

- «Напомни мне через 30 минут проверить почту»
- «Создай ежедневное напоминание в 9 утра: стендап» *(cron: `0 9 * * *`)*
- «Напоминай каждый будний день в 10:00: сделать отчёт» *(cron: `0 10 * * 1-5`)*
- «Покажи все активные напоминания»
- «Покажи только выполненные напоминания»
- «Дай сводку по напоминаниям»
- «Удали напоминание `<id>`»

## CLI-агент

В директории `cmd/agent/` находится интерактивный CLI-агент на базе Anthropic API (Haiku).
Агент подключается к MCP-серверу как subprocess и общается с пользователем в терминале.

```bash
# Собрать сервер и запустить агент
cd mcp-reminder
go build -o mcp-reminder.exe .
ANTHROPIC_API_KEY=your_key go run ./cmd/agent/
```

Агент поддерживает **push-уведомления**: когда напоминание срабатывает, сервер отправляет
нотификацию и агент сам (без ввода пользователя) вызывает `list_reminders` и сообщает
о результате.

## Описание MCP-инструментов

### `create_reminder`

Создаёт новое напоминание.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `title` | string | да | Текст напоминания |
| `due_in_minutes` | int | нет | Через сколько минут сработать |
| `cron_expr` | string | нет | Cron-расписание (5 полей) |

Нужно указать хотя бы один из `due_in_minutes` или `cron_expr`.

Пример ответа:
```json
{
  "message": "Напоминание создано: \"проверить почту\", сработает в 2025-03-11 21:00:00",
  "reminder": {
    "id": "a1b2c3d4...",
    "title": "проверить почту",
    "due_at": "2025-03-11T21:00:00Z",
    "status": "pending",
    "created_at": "2025-03-11T20:30:00Z"
  }
}
```

---

### `list_reminders`

Возвращает список напоминаний с опциональной фильтрацией.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `status` | string | нет | `pending` / `fired` / `cancelled` / пусто (все) |

Пример ответа:
```json
{
  "count": 2,
  "reminders": [...]
}
```

---

### `get_summary`

Возвращает агрегированную сводку. Параметров нет.

Пример ответа:
```
Напоминания:
  Всего: 5
  Ожидают: 3
  Сработали: 2
  Отменены: 0

Ближайшие напоминания:
  [a1b2c3d4] "стендап" → 2025-03-12 09:00
  [e5f6g7h8] "проверить почту" → 2025-03-12 10:30

Последние сработавшие:
  [x1y2z3w4] "дейли-митинг" — сработало в 2025-03-11 09:00
```

---

### `delete_reminder`

Отменяет напоминание по ID.

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| `id` | string | да | ID напоминания |

Пример ответа:
```json
{
  "message": "Напоминание \"стендап\" отменено",
  "id": "a1b2c3d4..."
}
```

## Как это работает внутри

### Жизненный цикл напоминания

```
create_reminder → status: pending
                       ↓
              Scheduler.checkAndFire()
              (каждые 30 секунд)
              due_at <= now?
                  ↓ да
              status: fired, fired_at: now
                  ↓
              cron_expr != ""?
                  ↓ да
              reschedule() → новое pending с следующим due_at
```

### Как планировщик проверяет задачи

Каждые 30 секунд планировщик:
1. Вызывает `store.GetDueReminders()` — SQL запрос: `WHERE status='pending' AND due_at <= now`
2. Для каждого найденного: `store.MarkFired(id)` — обновляет статус и fired_at
3. Вызывает коллбэк `OnFired` — сервер добавляет ресурс `reminder://fired/{id}`, go-sdk рассылает `notifications/resources/list_changed`
4. Если у напоминания есть `cron_expr` — вычисляет следующее время и создаёт новое напоминание

### Как периодические напоминания пересоздаются

После срабатывания периодического напоминания `reschedule()`:
1. Парсит `cron_expr` через `robfig/cron`
2. Вычисляет следующее время: `schedule.Next(time.Now())`
3. Создаёт новый `Reminder` с тем же `title` и `cron_expr`, но новым `ID` и `DueAt`

Каждое вхождение — отдельная запись в БД. Это позволяет видеть историю всех срабатываний.

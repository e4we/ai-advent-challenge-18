# CLAUDE.md

Этот файл содержит инструкции для Claude Code (claude.ai/code) при работе с кодом в этом репозитории.

## Сборка и тестирование

Все команды выполняются из поддиректории `mcp-reminder/`:

```bash
cd mcp-reminder
go build ./...                    # скомпилировать всё
go build -o mcp-reminder.exe .    # собрать бинарник (Windows)
go test ./... -v                  # запустить все тесты с подробным выводом
go test ./storage/... -v          # запустить тесты одного пакета
go test -run TestGetSummary ./storage/...  # запустить один тест
go vet ./...                      # статический анализ
go test ./... -race               # детектор гонок (требует CGo, недоступен с modernc.org/sqlite)
```

## Архитектура

MCP-сервер напоминаний на Go — stdio-сервер, позволяющий Claude создавать, отслеживать и отменять напоминания, хранящиеся в SQLite.

### Структура пакетов

- `models/` — типы данных `Reminder` и `ReminderSummary`, генерация ID через `crypto/rand`
- `storage/` — слой персистентности SQLite (`Store`), CRUD + агрегирующие запросы, использует `modernc.org/sqlite` (чистый Go, без CGo)
- `scheduler/` — фоновая горутина с тикером (30с по умолчанию, 100мс в тестах), обрабатывает просроченные напоминания и перепланирует cron-based через `robfig/cron/v3`
- `tools/` — обработчики MCP-инструментов, регистрируются через дженерик-функцию `mcp.AddTool` из `github.com/modelcontextprotocol/go-sdk`
- `main.go` — связывает всё воедино: Store → Scheduler (горутина) + MCP Server (stdio, блокирующий)

### Поток данных

```
Claude Desktop/Code ←→ MCP Server (stdin/stdout JSON-RPC)
                           ↓
                     Tool handlers → Store (SQLite)
                                       ↑
                     Scheduler (горутина, тикер) ──┘
```

### Ключевые архитектурные решения

- **Soft-delete при отмене**: `Store.Cancel()` ставит `status='cancelled'` (UPDATE, не DELETE). `Store.Delete()` по-прежнему существует для физического удаления.
- **`DueInMinutes` — это `*int`**: указатель позволяет различить «поле не передано» (nil) и «прямо сейчас» (0).
- **SQLite прагмы**: WAL-режим для конкурентного чтения/записи + `busy_timeout=5000` для предотвращения SQLITE_BUSY.
- **`GetSummary()` использует транзакцию**: 3 запроса обёрнуты в `tx.Begin()`/`tx.Commit()` для консистентного снимка данных.
- **Тесты планировщика используют polling** (хелпер `waitFor`), а не `time.Sleep`, чтобы избежать flaky-тестов на CI.

### Жизненный цикл напоминания

`pending` → (планировщик срабатывает когда `due_at <= now`) → `fired` → если `cron_expr != ""`, создаётся новое `pending`-напоминание со следующим временем.

`pending` → (пользователь отменяет) → `cancelled` (soft-delete, остаётся в БД для истории и сводки).

## Соглашения

- Все комментарии и пользовательские строки — на русском языке.
- Логи пишутся в stderr (stdout зарезервирован для MCP JSON-RPC протокола).
- В каждом тестовом файле есть хелпер `newTestStore(t)`, использующий `t.TempDir()` для изоляции.
- Обработчики инструментов тестируются напрямую (вызов функций-обработчиков), без MCP-сервера.
